### Weekly Summary

During Week 11, I focused on performing comprehensive testing of the **Pet Resort & Care System** after completing the system integration and deployment architecture configuration. I created a checklist of the main functions, tested each business workflow, and evaluated the communication between the user interface, Spring Boot application, and Amazon RDS database.

The testing process helped me identify several issues related to connectivity, data processing, response time, and environment configuration. I investigated the causes of these issues, adjusted incorrect settings, and performed additional testing to ensure that the system functions operated according to the project requirements.

In addition to functional testing, I used **Amazon CloudWatch** to monitor resource status, CPU utilization, application logs, and system errors. Analyzing these metrics and logs helped me understand the operational condition of the application and quickly identify problems that could affect system performance and stability.

I also reviewed the deployment configuration, database queries, and application response time. Several settings were adjusted to reduce unnecessary resource usage, improve request processing, and support more stable system operation.

Furthermore, I configured **AWS Budgets** to track the cost of AWS services and created budget alert thresholds. This allowed me to manage the resources more effectively, identify services that could generate unexpected costs, and make appropriate adjustments when necessary.

By the end of Week 11, the system had been thoroughly tested, the main issues had been resolved, and the operational configurations had been further optimized. Through these activities, I improved my skills in system testing, log analysis, AWS resource monitoring with Amazon CloudWatch, cost management with AWS Budgets, performance optimization, and cloud troubleshooting.
